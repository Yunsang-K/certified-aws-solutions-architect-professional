# Snow family

* Snowball 계열은 **대용량 데이터를 AWS로 가져오거나(AWS로 이전) AWS에서 내보내기(반출)** 위해 설계된 서비스/디바이스들입니다.
* Snow 계열 제품은 **물리적 스토리지 장비(가방 형태 디바이스, 트럭/컨테이너 형태)** 입니다.
* 보통 **빈 장비를 배송받아 데이터를 적재한 뒤 AWS로 반송**하거나, 반대로 **AWS에서 데이터를 적재해 보내주는 방식**으로 사용합니다.

---

## Snowcone

* **휴대 가능**, **견고**, **보안 강화**된 **엣지 컴퓨팅 + 데이터 전송** 디바이스입니다.
* Snowcone은 2가지 모델이 있습니다.

  * **Snowcone**: 2 vCPU, 4GB 메모리, **8TB HDD**
  * **Snowcone SSD**: 2 vCPU, 4GB 메모리, **14TB SSD**
* 주요 사용 사례

  * 엣지 위치에서 데이터를 **수집 → 로컬 처리(즉시 인사이트) → 이후 AWS로 전송**
  * 공장/엣지 환경에서 센서/장비가 생성하는 데이터를 **지속적으로 수집**해 AWS로 전송
  * AWS 스토리지에 있는 미디어/과학 데이터 등을 **파트너·고객에게 배포**
  * 여러 엣지 위치의 미디어/과학 데이터를 **집계(모아서) AWS로 전송**

---

## Snowball

* AWS에서 **Job을 생성/등록**하면 디바이스를 배송해주고, 적재 후 반송하는 장비입니다.
* Snowball에 저장된 데이터는 **KMS 기반 암호화**가 적용됩니다.
* 디바이스 유형(용량): **50TB / 80TB**
* 네트워크 연결 옵션

  * **1Gbps** (RJ45 1GBase-TX)
  * **10Gbps** (LR/SR)
* 경제적인 데이터 전송 범위: 대략 **10TB ~ 10PB**
  (필요 시 **여러 대**를 사용해 확장)
* 여러 대를 주문해 **여러 사업장(다중 사이트)** 으로 각각 배송할 수 있습니다.
* Snowball은 **스토리지 전용(컴퓨팅 기능 없음)** 입니다.

---

## Snowball Edge

* **스토리지 + 컴퓨팅(EC2 등)** 기능을 함께 제공하는 장비입니다.
* 클래식 Snowball보다 **더 큰 용량**, **더 빠른 네트워크**를 제공합니다.
* 3가지 타입

  * **Storage Optimized (EC2 가능)**: 80TB, 24 vCPU, 32GiB RAM, EC2용 로컬 SSD 1TB
  * **Compute Optimized**: 100TB + 7.68 NVMe, 52 vCPU, 208GiB RAM
  * **Compute Optimized with GPU**: 100TB + 7.68 NVMe, 52 vCPU, 208GiB RAM, GPU 포함
* 데이터 유입 시점에 **현장에서 바로 처리(전처리/분석/필터링 등)** 가 필요하거나, **원격 사이트**에 적합합니다.

---

## Snowmobile (중단됨)

* 트럭에 실린 **컨테이너 형태의 이동식 데이터센터**입니다.
* AWS에 **특별 주문** 형태로 진행됩니다.
* 단일 사이트에서 **10PB 이상** 같은 초대용량 이전에 적합합니다.
* Snowmobile 1대당 **최대 100PB** 저장 가능
* **다중 사이트** 또는 **10PB 미만** 규모에는 비경제적입니다.
